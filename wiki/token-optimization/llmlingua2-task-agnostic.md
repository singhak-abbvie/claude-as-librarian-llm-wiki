# LLMLingua-2: Task-Agnostic Compression

**Source:** ai-research/llmlingua2-paper.md, ai-research/llmlingua-github-repo.md

LLMLingua-2 represents a major advancement in prompt compression, using data distillation from GPT-4 and bidirectional context understanding to achieve faster, more generalizable compression than its predecessor.

## Key Takeaways

- **3x-6x faster** than original LLMLingua
- Uses bidirectional context (BERT-style) instead of unidirectional (GPT-style)
- Formulates compression as token classification rather than entropy-based removal
- Trained via data distillation from GPT-4 for better task-agnostic performance
- Superior out-of-domain generalization compared to LLMLingua
- Achieves **1.6x-2.9x latency acceleration** with 2x-5x compression ratios

## The Problem with Information Entropy

Previous approaches (including LLMLingua) used information entropy to identify removable tokens. This has limitations:

1. **Unidirectional context**: Only considers tokens before the current position
2. **Misaligned objective**: Entropy doesn't directly optimize for compression quality

## LLMLingua-2 Solution

### Token Classification Approach
Instead of using entropy, LLMLingua-2 treats compression as a **binary token classification problem**:
- Each token is classified as "keep" or "remove"
- This ensures faithfulness to the original prompt

### Bidirectional Context
Uses Transformer encoder architecture (XLM-RoBERTa, mBERT) to capture context from both directions, identifying essential information more accurately.

### Data Distillation
Training data is generated by having GPT-4 identify which tokens are essential vs. removable, then this knowledge is distilled into a smaller model.

## Performance Comparison

| Aspect | LLMLingua | LLMLingua-2 |
|--------|-----------|-------------|
| Context | Unidirectional | Bidirectional |
| Model size | LLaMA-7B (14GB) | BERT-level (few GB) |
| Speed | Baseline | 3-6x faster |
| Out-of-domain | Good | Better |
| Task-agnostic | Limited | Strong |

### Benchmark Results

Evaluated on MeetingBank, LongBench, ZeroScrolls, GSM8K, and BBH:
- Significant performance gains over baselines
- Robust generalization across different LLMs
- 1.6x-2.9x latency acceleration at 2x-5x compression

## Usage

```python
from llmlingua import PromptCompressor

# Using the larger model
llm_lingua = PromptCompressor(
    model_name="microsoft/llmlingua-2-xlm-roberta-large-meetingbank",
    use_llmlingua2=True,
)

# Or using the smaller, faster model
llm_lingua = PromptCompressor(
    model_name="microsoft/llmlingua-2-bert-base-multilingual-cased-meetingbank",
    use_llmlingua2=True,
)

compressed = llm_lingua.compress_prompt(
    prompt, 
    rate=0.33,  # Target 33% of original
    force_tokens=['\n', '?']  # Always keep these tokens
)
```

## Key Parameters

| Parameter | Purpose |
|-----------|---------|
| `rate` | Target compression ratio (e.g., 0.33 = 33% of original) |
| `force_tokens` | Tokens to always preserve (e.g., punctuation) |
| `use_llmlingua2` | Enable LLMLingua-2 mode |

## When to Use LLMLingua-2

**Choose LLMLingua-2 when:**
- Speed is critical
- GPU memory is limited
- Processing diverse content types
- Need task-agnostic compression
- Working with out-of-domain content

**Consider original LLMLingua when:**
- Maximum compression ratio is priority (20x vs 5x)
- Working within known domains
- Already have large GPU available

## Technical Innovation

### Why Bidirectional Context Matters

In the sentence "The quick brown fox jumps over the lazy",  identifying that "brown" can be removed requires understanding:
- What comes before: "The quick"
- What comes after: "fox jumps over the lazy"

Unidirectional models miss the "after" context, potentially removing important words.

### Data Distillation Process

1. **Annotation**: GPT-4 labels tokens as essential/removable in training examples
2. **Training**: Small encoder model learns to replicate GPT-4's judgments
3. **Inference**: Trained model rapidly classifies tokens in new prompts
4. **Compression**: Remove tokens classified as non-essential
